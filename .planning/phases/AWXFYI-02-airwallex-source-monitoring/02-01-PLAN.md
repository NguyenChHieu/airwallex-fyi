# Plan 02-01: Source Discovery and Configuration

## Goal

Build the verified Airwallex source discovery layer for public Blog and Newsroom article URLs, with configurable sitemap and first-run seed settings.

## Scope

This plan only discovers candidate article URLs from the public sitemap. It does not fetch article pages, persist posts, summarize posts, or trigger notifications.

## Artifacts This Phase Produces

- `AppProperties` source settings for sitemap URL and first-run seed limit.
- A small HTTP abstraction for public Airwallex fetches.
- A sitemap discovery service returning typed article candidates.
- A sitemap fixture covering accepted and rejected URL shapes.
- Unit tests for config binding and sitemap filtering.

## Must Haves

### Truths

- D-01: Accept only strict global article paths: `/global/blog/{slug}` and `/global/newsroom/{slug}`.
- D-02: Exclude pagination URLs, regional duplicates, non-article URLs, and private app/API paths.
- D-03: Determine `sourceType` from URL path first, with extraction allowed to confirm or enrich later.
- D-04: If sitemap fetch or parsing fails, the monitor must stay usable, report discovery failure, and avoid writing new posts.
- Live verification on 2026-06-20 showed `https://www.airwallex.com/global/sitemap-blog.xml` returned HTTP 200 with public Blog and Newsroom URLs plus pagination URLs that must be filtered out.
- Live robots.txt verification on 2026-06-20 showed public crawling is allowed while private app/API paths are disallowed.

### Constraints

- Use Spring `RestClient`, not `RestTemplate`.
- Add `org.jsoup:jsoup:1.22.2`; this version was verified from Maven Central metadata on 2026-06-20.
- Keep the source layer fakeable in tests; no live network calls in tests.
- Do not infer unverified URL shapes beyond the strict filters above.

- Accept only exact URL path matches for:
  - `https://www.airwallex.com/global/blog/{slug}`
  - `https://www.airwallex.com/global/newsroom/{slug}`

- Preserve parsed `<lastmod>` as `sitemapLastmod` when present; use `null` when absent.
</action>

- Discovery errors are not converted into successful empty results.
</acceptance_criteria>

## Threat Model

<threat_model>
- Risk: accidentally crawling private app/API paths. Mitigation: strict URL path allowlist and fixture coverage for rejected examples.
- Risk: sitemap shape changes cause silent empty discovery. Mitigation: parsing failures surface to the monitor as discovery failures instead of successful empty runs.
- Risk: tests depend on live Airwallex availability. Mitigation: use local fixtures and fake HTTP clients only.
</threat_model>
