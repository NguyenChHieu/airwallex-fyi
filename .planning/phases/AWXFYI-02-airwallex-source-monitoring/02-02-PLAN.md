# Plan 02-02: Article Extraction and Content Hashing

## Goal

Extract usable article data from verified Airwallex Blog and Newsroom page shapes, normalize rich text into plain text, and compute stable content hashes for later dedupe/update behavior.

## Scope

This plan extracts data from one article page at a time. It does not discover sitemap URLs, persist records, call AI models, or notify users.

## Artifacts This Phase Produces

- Article extraction DTOs and extraction service.
- Rich-text flattening for Contentful-style `__NEXT_DATA__` article content.
- HTML fallback extraction for degraded structured data cases.
- Stable SHA-256 content hashing from normalized title, description, and body.
- Blog, Newsroom, and fallback fixtures with tests.

## Must Haves

### Truths

- D-05: A usable article requires title and meaningful body. Date, description, and author are optional.
- D-06: Structured `__NEXT_DATA__` extraction comes first. Verified Blog path is `pageData.post.fields`; verified Newsroom path is `pageData.pr.fields`.
- D-07: HTML fallback is only used when structured content is missing or unparseable; fallback success must be visible in logs or run results.
- D-08: Parser rules must be backed by live checks or captured fixtures. Do not assume page shapes.
- D-09: Store normalized plain text as `article_body`.
- D-10: Image URLs may be extracted opportunistically, but image persistence is not required in Phase 2.
- D-11: Content hash is based on normalized title, description, and body; noisy metadata/media are excluded.
- D-12: Missing dates become `published_at = null`; preserve sitemap lastmod separately in persistence.
- D-13: Keep representative Blog and Newsroom fixtures; do not use live network tests.
- Live verification on 2026-06-20 confirmed Blog uses `props.pageProps.pageQuery.pageData.post.fields` and Newsroom uses `props.pageProps.pageQuery.pageData.pr.fields`.

### Constraints

- Use Jsoup to parse HTML and select `script#__NEXT_DATA__`.
- Use Jackson tree parsing for JSON; avoid regex-based JSON extraction.
- Keep fixtures representative but small enough for readable tests.
- No OpenAI or notification integration in this phase.

## Threat Model

<threat_model>
- Risk: `__NEXT_DATA__` path drift creates false summaries from wrong page data. Mitigation: extractor supports only verified paths and tests fixture paths explicitly.
- Risk: regex parsing corrupts JSON or skips escaping. Mitigation: Jsoup selects the script and Jackson parses the JSON tree.
- Risk: stored body contains noisy navigation/footer text. Mitigation: structured extraction is preferred; fallback uses constrained semantic selectors and requires meaningful body text.
</threat_model>
