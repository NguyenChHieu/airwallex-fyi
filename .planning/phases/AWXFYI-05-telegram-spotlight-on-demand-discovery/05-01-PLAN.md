# Plan 05-01: Telegram Spotlight Vertical Slice

## Goal

Add a useful on-demand discovery loop to the Telegram bot while preserving the centralized post/summary model and scheduled digest semantics.

## Tasks

### Task 1: Implement Spotlight selection and canonical summary reuse/backfill

<read_first>
- `src/main/kotlin/com/airwallexfyi/posts/PostRecord.kt`
- `src/main/kotlin/com/airwallexfyi/articles/ArticleExtractor.kt`
- `src/main/kotlin/com/airwallexfyi/monitor/PostStateService.kt`
- `src/main/kotlin/com/airwallexfyi/summaries/ArticleSummaryService.kt`
- `src/main/kotlin/com/airwallexfyi/digests/DigestEligibilityService.kt`
- `.planning/phases/AWXFYI-05-telegram-spotlight-on-demand-discovery/05-CONTEXT.md`
</read_first>

<action>
- Add `SpotlightService` under `com.airwallexfyi.spotlights`.
- Build a bounded 25-post candidate pool sorted by `publishedAt`, then `sitemapLastmod`, then `discoveredAt`, and choose one candidate randomly.
- Reuse a stored summary when the post does not require content-change approval.
- Convert a stored post body into `ExtractedArticle`, or call `ArticleExtractor` for only the selected URL when the body is missing.
- Add a narrow `PostStateService` operation that hydrates the existing post's canonical article fields without changing `processing_status`.
- Generate/persist a missing summary with `ArticleSummaryService`; replace a stale summary only for `APPROVAL_NEEDED`.
- Preserve `SEEDED`/`BASELINED`; transition recoverable current-post states to `SUMMARY_READY` after successful generation.
- Format the Spotlight message and bounded failure fallback with the direct URL.
</action>

<acceptance_criteria>
- Existing summary path makes zero AI and article-fetch calls.
- Missing-summary path persists exactly one summary.
- Missing-body path persists extracted canonical content.
- Baseline backfill leaves the post `BASELINED` and ineligible for daily digest selection.
- Failure response includes the selected post URL.
</acceptance_criteria>

<acceptance_criteria>

- A duplicate webhook update does not produce duplicate Spotlight replies.
- Existing Telegram command tests remain green.
</acceptance_criteria>
