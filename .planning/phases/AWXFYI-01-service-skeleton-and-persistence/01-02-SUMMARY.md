## Decisions Made

- Used explicit `@Column` annotations for every persisted property so Spring Data JDBC does not rely on runtime naming-strategy behavior.
- Used `Persistable<UUID>` so application-generated UUIDs are treated as new records during inserts.
- Kept the persistence test as a Spring Boot test with Flyway enabled so the actual migration is exercised.

## Deviations from Plan

- Added `spring-boot-starter-data-jdbc` to `build.gradle.kts`. The generated scaffold had `spring-boot-starter-jdbc`, but repository interfaces extending `CrudRepository` require Spring Data JDBC.
